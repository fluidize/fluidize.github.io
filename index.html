  • Genetic Algorithms for Algorithmic Trading

    Using python abstract syntax trees to represent genes, algorithms can be randomly generated to process logic between technical indicators and output trading signals. An individual consists of a minimum of one indicator gene, two logic genes, and one signal gene to interpret logic. Logic is arguably the most important gene and is able to compare two indicators(i2i), an indicator to the price(i2p), or compare an indicator to a constant value(i2c). The current implementation uses the following in order: population generation, bayesian optimization(optuna), and staged pruning to find the best performing algorithm within a generated population.

  • Jupiter-Based Live Trading System

    Through the Jupiter Decentralized Exchange API, this system takes in an algorithm(any python function) and asynchronously feeds market data to it. Market data is streamed from sources such as binance, and trades made by the algorithm are processed through the Jupiter Aggregator. Features a live paper trading mode for forward testing. Latency and small price mismatches(between data source and DEX) are expected.

  • Multiheaded Neural Network for Optimal Asset Allocation

    This model uses two heads in a residual-like architecture, both trained on the same data with separate objectives. The distribution head estimates the forward velocity of the asset according to a student-t distribution. The allocation head uses both the distribution and the input data to estimate the optimal percentage allocation of the asset, according to the given loss function.

  • BirdCLEF 2026 🐦

    Starting from a Mel-scaled spectrogram, the efficientnet-b0 Convolutional Neural Network(with pretrained weights for faster convergence) is used as the model backbone to generate convoluted embeddings. To efficiently reduce embedding dimensions, multi-scale adaptive average pooling combined with attention pooling give the main classifier network a widened perspective on the bird sounds. In intermediary steps(between embedding and classification), GLUs(gated linear units) are used to create a "focusing" effect, allowing the network to learn which features to prioritize without using expensive self-attention. It is crucial that the GLU weights are initialized close to 1, otherwise the randomly generated weights may interfere with learning. Although performing below distilled Perch-based models, combining this architecture with time+frequency masking and temperature scaling gave a final score of 0.79241!
